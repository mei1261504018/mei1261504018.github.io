我的第一篇博客

我的第一篇博客–关于怎么搭建静态博客hexo

1.安装前的工具、环境准备

-git         --是一个网上仓库     
-node.js      --Node 是一个让 JavaScript 运行在服务端的开发平台（我理解就是一个html环境）
-hexo          --是一个轻量级博客框架

安装步骤：

####### 步骤1：nodej.js 下载并安装

node官方链接地址 :https://nodejs.org/en/ 网页打开下载，并傻瓜式安装

安装完成后有两个组件
node.js 文件本身
npm  包管理器--搭建博客必须的东西

####### 步骤2：npm环境配置
查看node版本： node -v
查看npm版本： npm -v

####### 步骤3：安装cnpm
npm是针对国外地址的，速度慢。通过npm安装一个国内的安装镜像工具cpm

>npm install -g cnpm --registry=http://registry.npm.taobao.org

安装成功后，检测cnpm

>cnpm -v

####### 步骤4:用cnpm安装进行安装hexo
>cnpm install -g hexo-cli
安装成功后，验证是否成功 

>hexo -v

出现版本号，及验证成功

2.开始搭建博客系统

1.新建一个文件夹“mblog” （文件夹名字随便），这个文件夹的作用就是，以后的hexo所有的文件。包括hexo配置设置、博客内容的储存等。出现错误可以删除文件夹，重现来

方式一:mkdir 文件名   通过cmd窗口
方式二：自己在文件夹任意位置。

2.初始化一个博客

打开cmd窗口，进入之前新键的“blog”包下，进行网上资源的下载

>hexo init

成功后打开文件夹，问价夹下就会有下载好的hexo框架的相关内容

3、启动hexo博客的预览模式

>hexo s:

-端口号启动,网页打开端口号，就会看见默认生成的博客文章

3.编写、更新维护博客内容

方式一：在blog目标文件夹下，找到\source\_posts位置。
_post 文件夹下的以.md后缀名生成的文件就是自己的博客。
新建以.md文件，用note++、markdoem文件编辑器编辑，保存成功


方式二：控制台输出 
新建一个文档标题，内容为空打开文档，在tags下编辑内容。
>hexo n “文章标题”
注：(停止掉hexo CTRL+c退出预览模式后再进行编辑，否则控制台输出不了)

更新博客

重新生成博文文件

控制台博文清理： 

>hexo clean

博文生成：

>hexo g（generate）

hexo重现启动：

>hexo s

-        然后默认网址输出就可以了

功能强化1—部署到免费的服务器

搭建到免费的平台上
为什么要搭建。在我们自己未搭建时，每次使用都需要hexo s命令部署.当搭建到免费的平台仓库后，可以直接打开地址访问博客。 这里访问的博客内容是最后一个部署的内容，如果想更新就得更新部署

github平台的部署

————————————————————————————————

步骤1 : 构建仓库

1、登录GitHub官网，新建仓库。仓库名称“自己的昵称”.github.io ,作为一个空仓库。用来保存后期上传hexo文件

步骤2 : git环境搭建

在cmd窗口的hexo文件夹目录下安装一个部署git的 命令行：

>cnpm install --save hexo-deployer-git

安装成功后

步骤3 : 设置_config.yml文件的三个标签:

这个_config.yml文件 在之前自己设置mblg文件夹下

1、type：git     -类型
2、repo:            -git仓库的地址
3、breach:        -默认

步骤4 : 部署到远端

>gexo d(deployer-部署)

步骤5 : 刷新、浏览器访问

这里的浏览器地址就是之前参考设置的地址

下载成功后，进入yilia文件夹。找到_config.yml文件找打theme标签修改默认 theme：yilia

3、开始配置到自己文件下，修改config.yml文件,,,,找到theme标签下 theme：yilia
